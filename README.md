# Personal Newspaper

A private, daily, evidence-backed newspaper generated by Codex and rendered by a stable Sites reader. The outside page contains a useful report, not a teaser; opening a story adds detail and original-source links. Love/less reactions influence later editions through non-sensitive editorial metadata.

## Architecture

- `skills/personal-newspaper/` contains the Agent Skills-compliant workflow, print grammar, deterministic commands, template, and evals.
- `app/EditionReader.tsx` renders immutable edition bundles and keeps reactions inside each article package.
- D1 stores the owner profile, edition index, reaction signals, revision state, and share capabilities.
- R2 stores immutable edition bundles.
- `app/api/agent/` is the bearer-authenticated Codex publication surface.
- `.openai/hosting.json` declares the Sites project and bindings.

## Local development

Requires Node.js `>=22.13.0` and pnpm.

```bash
pnpm install --frozen-lockfile
pnpm dev
```

## Pipeline commands

Run commands from this directory. Keep `AUTOMATION_TOKEN` secret.

```bash
pnpm edition:context -- --output "$EDITION_CONTEXT" --url "$PAPER_URL"
pnpm edition:prepare -- --draft "$EDITION_DRAFT" --context "$EDITION_CONTEXT" --output "$EDITION_BUNDLE"
pnpm edition:validate -- --file "$EDITION_BUNDLE"
pnpm edition:publish -- --file "$EDITION_BUNDLE" --url "$PAPER_URL"
pnpm edition:restore -- --id "$PREVIOUS_EDITION_ID" --expected-current "$FAILED_EDITION_ID" --url "$PAPER_URL"
```

Read [the skill](skills/personal-newspaper/SKILL.md) before operating the pipeline. Drafts follow `skills/personal-newspaper/assets/edition-template.json` and omit `generation`; `edition:prepare` binds the exact private context snapshot.
